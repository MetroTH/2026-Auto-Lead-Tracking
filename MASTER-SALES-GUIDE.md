## ⚠️ ทำไมต้องระวังการวาง

ไฟล์กลางถูกอ่าน **2 วิธีที่ต่างกัน** ต้องรักษาไว้ทั้งคู่:

| โปรเจกต์ | อ่านคอลัมน์แบบ | ผลถ้าทำผิด |
|---|---|---|
| **2026-Auto-Lead** (invoice + quotation) | **ตำแหน่งตายตัว** (index) เช่น `Invoice No = คอลัมน์ B`, `Total Sales = K`, `Vertical = AB` | เลื่อนคอลัมน์ = ตัวเลขผิดทั้งหมด **แบบไม่ error** |
| **Loyalty-Point** | **ชื่อหัวคอลัมน์** (หา `invoice no` / `point` / …) | เปลี่ยนชื่อหัว = ค่าหาย / เป็น 0 |

รวมกันแล้วกฎเดียวที่ปลอดภัยคือ: **รักษาทั้งลำดับคอลัมน์และชื่อหัวไว้เหมือนเดิม**

## ✅ ทำได้ (ปลอดภัย)

1. **วางทับข้อมูลทั้งแทป** — ลบข้อมูลเก่า (row 2 ลงไป) แล้ววางชุดใหม่ตั้งแต่ `A2`
   ทุกโปรเจกต์ Full Sync จะ rebuild ใหม่หมด
2. **วางต่อท้าย (append) ก็ได้** — ต่อให้แถวซ้ำ/ช่วงวันซ้อนกัน โค้ดตัดซ้ำเองด้วย key
   (invoice = InvNo+วันที่+รหัส / quotation = Quotation+PartNo+Value+วันที่+DCN / Loyalty = InvNo)
3. **วางถี่แค่ไหนก็ได้** — วันละครั้ง / สัปดาห์ละครั้ง ทุกโปรเจกต์อ่าน snapshot ล่าสุด
4. **เพิ่มคอลัมน์ใหม่ต่อท้ายขวาสุด** (หลังคอลัมน์สุดท้าย) — ปลอดภัย ไม่ดันตำแหน่ง index เดิม
5. วาง 2 แทป (`Raw Invoice` / `Raw Quotation`) แยกกันกี่รอบก็ได้ อิสระต่อกัน

## ⛔ ห้ามทำ (ข้อมูลจะเพี้ยนแบบเงียบ ๆ)

1. **ห้ามสลับ / แทรก / ลบคอลัมน์ตรงกลาง** ← อันตรายที่สุด
   → 2026 อ่านด้วยเลขคอลัมน์ เลื่อนแม้ 1 ช่อง `Total Sales` จะไปหยิบ Credit Term,
   `Vertical` จะหยิบคอลัมน์อื่น — **ไม่ error แต่ผิดทั้งหมด**
2. **ห้ามเปลี่ยนชื่อหัวคอลัมน์** (row 1) — โดยเฉพาะ `Invoice No`, `Total Sales`, `Point`,
   `รหัสลูกค้า / DCN`, `Vertical` → Loyalty จะหาไม่เจอ
3. **ห้ามเปลี่ยนชื่อแทป** — ต้องเป๊ะ `Raw Invoice` / `Raw Quotation`
   (โค้ดหาแทปด้วย `getSheetByName` ผิดตัวอักษร/เว้นวรรค = error)
4. **ห้ามลบ / ทับแถวหัว (row 1)** — ทุกโปรเจกต์ข้าม row แรกเป็น header
5. **ห้ามเอา invoice กับ quotation ปนแทปเดียวกัน** — คนละโครงสร้างคอลัมน์
6. **ห้ามวางเป็นสูตร / IMPORTRANGE** — ให้ **Paste values only** (Ctrl+Shift+V)
   กันวันที่/เลข 0 นำหน้าเพี้ยน และกันสูตรพังเมื่อไฟล์ต้นทางเปลี่ยน

---

## 💡 ขั้นตอนวางที่แนะนำ (จำ 3 ข้อ)

1. ดาวน์โหลด raw → เปิดไฟล์กลาง → **เลือกแทปให้ถูก** → คลิก `A2` → **Paste values only** ทับข้อมูลเก่า
2. อย่าไปยุ่งกับ **ลำดับคอลัมน์ + แถวหัว** — เปลี่ยนแค่ "เนื้อข้อมูล" ข้างใน
3. วางเสร็จ ปล่อย trigger auto รัน หรือกด Full Sync เอง — ระบบตัดซ้ำ + กรอง 4 เดือนให้เอง

> **จำง่าย ๆ:** *เปลี่ยนได้แค่ข้อมูลข้างใน — โครง (หัวคอลัมน์ + ลำดับ + ชื่อแทป) ห้ามแตะ*

## หมายเหตุเรื่องจำนวนแถว (Full Sync vs Incremental)

- **Full Sync** = ล้างแทปแล้วสร้างใหม่ทั้งหมด → ยอดจริง ณ วันนั้น (สะอาด)
- **Incremental** (trigger รายวัน) = ต่อท้ายเฉพาะแถวใหม่ ไม่เคยล้างของเก่า → ยอดสะสม

ดังนั้นถ้ายอดแถวจาก Full Sync **น้อยกว่า** ยอดเดิมที่สะสมมานาน = **ปกติ** (ของเดิมบวมจากการสะสม)
Full Sync คือยอดที่ถูกต้อง
